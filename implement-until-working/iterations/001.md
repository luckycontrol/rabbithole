# Iteration 001

## Summary
- Status: progress
- Goal: Prevent composing Enter from submitting Korean follow-up text in both reader and card composers.
- Why this iteration: Both follow-up keydown handlers submitted every unshifted Enter and did not inspect the browser's IME composition signal.
- Method: Event-level `KeyboardEvent.isComposing` guard at both follow-up submission boundaries.

## Context Read
- `implement-until-working/task.md` reviewed: yes (created before production edits)
- `implement-until-working/method-history.md` reviewed: yes (created before production edits)
- Prior iteration files reviewed: none; this is iteration 001
- Design reviewed: `docs/plans/2026-07-14-followup-ime-design.md`

## Hypothesis
The IME confirmation Enter is reaching the ordinary follow-up keydown handlers. Requiring `!event.isComposing` will leave that Enter to the IME, retain the draft, and allow the later ordinary Enter to submit the complete textarea value once.

## Changes Made
- Added a real Chromium regression test for Korean composition in the reader and per-card follow-up composers.
- Added `!e.isComposing` to the reader follow-up Enter condition.
- Added the same guard to the card follow-up Enter condition.
- Rebuilt the committed live and frozen UI bundles.
- Added the stable task, design, method history, and this iteration handoff.
